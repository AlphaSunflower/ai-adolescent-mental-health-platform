# 书籍模块接口文档

## 一、概述

本文档描述了书籍模块的后端接口，供前端开发人员参考使用。

### 1.1 接口基础信息

- **基础URL**: `http://localhost:8080`
- **数据格式**: JSON
- **编码**: UTF-8

### 1.2 通用响应格式

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | Integer | 状态码，200表示成功，500表示失败 |
| message | String | 提示信息 |
| data | Object | 返回数据 |

### 1.3 分页响应格式

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "total": 100,
    "records": [],
    "current": 1,
    "size": 10,
    "pages": 10
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| total | Long | 总记录数 |
| records | Array | 当前页数据列表 |
| current | Long | 当前页码 |
| size | Long | 每页数量 |
| pages | Long | 总页数 |

## 二、用户端接口

> 说明：用户端接口 `/book/**` 已配置为公开访问，无需登录即可访问。

### 2.1 获取书籍列表

获取上架的书籍列表，支持分页和关键词搜索。

```
GET /book/list
```

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | Integer | 否 | 页码，默认1 |
| size | Integer | 否 | 每页数量，默认10 |
| keyword | String | 否 | 搜索关键词（匹配书籍标题） |

**请求示例：**

```
GET /book/list?page=1&size=10&keyword=心理
```

**响应示例：**

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "total": 2,
    "records": [
      {
        "id": 1,
        "title": "青少年心理健康指南",
        "coverUrl": "https://xxx.com/cover.jpg",
        "description": "帮助青少年更好地理解和管理情绪，提升心理素质。",
        "address": "https://example.com/book/1",
        "viewCount": 1234,
        "commentCount": 56
      },
      {
        "id": 2,
```

## 三、超级管理员端接口

> 说明：管理员接口 `/admin/book/**` 需要超级管理员权限（role=4）。
> 需要在请求头中携带有效的 Authorization Token。

### 3.1 获取书籍管理列表

获取所有书籍的管理列表，支持分页、搜索和状态筛选。

```
GET /admin/book/list
```

**请求头：**

| 参数 | 必填 | 说明 |
|------|------|------|
| Authorization | 是 | Bearer Token，超级管理员登录凭证 |

**请求参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | Integer | 否 | 页码，默认1 |
| size | Integer | 否 | 每页数量，默认10 |
| keyword | String | 否 | 搜索关键词（匹配书籍标题） |
| status | Integer | 否 | 状态筛选（0-下架，1-上架） |

**请求示例：**

```
GET /admin/book/list?page=1&size=10&status=1
```

**响应示例：**

```json
{
  "code": 200,

  "data": {
    "total": 10,
    "records": [
      {
```

## 六、接口调用示例

### 6.1 使用 cURL

**获取书籍列表：**

```bash
curl -X GET "http://localhost:8080/book/list?page=1&size=10"
```

**发表评论（需登录）：**

```bash
curl -X POST "http://localhost:8080/book/comment" \
  -H "Authorization: Bearer your_token_here" \
  -H "Content-Type: application/json" \
  -d '{"bookId": 1, "content": "这本书很棒！"}'
```

**新增书籍（需超级管理员权限）：**

```bash
curl -X POST "http://localhost:8080/admin/book" \
  -H "Authorization: Bearer admin_token_here" \
  -H "Content-Type: application/json" \
  -d '{"title": "新书上架", "coverUrl": "https://xxx.com/cover.jpg", "status": 1}'
```

### 6.2 使用 JavaScript (Fetch API)

```javascript
// 获取书籍列表
fetch('http://localhost:8080/book/list?page=1&size=10')
  .then(response => response.json())
  .then(data => console.log(data));

// 发表评论
fetch('http://localhost:8080/book/comment', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer your_token_here',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    bookId: 1,
    content: '这本书很棒！'
  })
})
  .then(response => response.json())
  .then(data => console.log(data));
```

## 七、数据库表结构

### 7.1 book 表

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | BIGINT | 主键ID，自增 |
| title | VARCHAR(200) | 书籍标题 |
| cover_url | VARCHAR(500) | 封面图片URL |
| description | TEXT | 书籍简介 |
| address | VARCHAR(500) | 跳转地址 |
| view_count | INT | 浏览数，默认0 |
| comment_count | INT | 评论数，默认0 |
| sort_order | INT | 排序权重，默认0 |
| status | TINYINT | 状态（0下架 1上架），默认1 |
| create_time | DATETIME | 创建时间 |
| update_time | DATETIME | 更新时间 |
| deleted | TINYINT | 逻辑删除标记，默认0 |

### 7.2 book_comment 表

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | BIGINT | 主键ID，自增 |
| book_id | BIGINT | 书籍ID |
| user_id | BIGINT | 用户ID |
| content | TEXT | 评论内容 |
| create_time | DATETIME | 评论时间 |
| deleted | TINYINT | 逻辑删除标记，默认0 |
