# 小爱倾听多模态功能修复记录

**日期**: 2026-04-10
**问题**: 前端成功连接WebSocket，但控制台消息没有变化，无法正常对话

## 问题分析

### 系统架构

```
前端 (Vue) ─── WebSocket ────▶ 后端代理 (OmniRealtimeWebSocketProxy)
                                              │
                                              └── WebSocket ────▶ 阿里云 qwen3.5-omni-plus-realtime
```

### 发现的问题

#### 1. 音频发送字段错误

**原代码**:
```javascript
const message = {
  type: `${typePrefix}.append`,
  delta: chunk  // ❌ 错误
};
```

**正确格式**:
```javascript
const message = {
  type: "input_audio_buffer.append",
  audio: chunk  // ✅ 阿里云API要求使用 audio 字段
};
```

#### 2. 缺少 response.create 事件

**问题**: 使用 Manual 模式（禁用VAD）时，需要手动发送 `response.create` 事件来触发模型响应。

**修复**: 在 `sendAudioData` 函数末尾添加:
```javascript
const responseCreateMessage = {
  event_id: `event_${Date.now()}_response`,
  type: "response.create"
};
socket.send(JSON.stringify(responseCreateMessage));
```

#### 3. session.update 配置问题

**原配置**:
```javascript
turn_detection: {
  type: "server_vad",
  threshold: 0.5,
  silence_duration_ms: 800
}
```

**修复为 Manual 模式**:
```javascript
turn_detection: null
// 注意：不要添加 input_audio_transcription 字段
```

#### 4. 音频播放解码问题

**原代码**: 使用 `decodeAudioData` 解码原始PCM数据会失败

**修复**: 手动构建AudioBuffer，正确处理24kHz采样的PCM数据

```javascript
const pcmData = new Int16Array(bytes.buffer, bytes.byteOffset, bytes.byteLength / 2);
const audioBuffer = audioContext.createBuffer(1, pcmData.length, 24000);
const channelData = audioBuffer.getChannelData(0);
for (let i = 0; i < pcmData.length; i++) {
  channelData[i] = pcmData[i] / 32768.0;
}
```

## 常见错误排查

### 错误: EngineCore encountered an issue

这个错误表示阿里云服务端内部错误，可能原因：

1. **模型服务暂时不可用** - 等待几分钟后重试
2. **API Key 没有开通该模型** - 需要在阿里云百炼平台开通 qwen3.5-omni-plus-realtime 模型
3. **账户余额不足** - 检查阿里云账户余额

### 验证API Key权限

登录阿里云百炼控制台 (https://bailian.console.aliyun.com/)，检查：
- API Key 是否有效
- 是否已开通 qwen3.5-omni-plus-realtime 模型
- 账户是否有足够的调用额度

## 延迟优化（2026-04-10）

### 问题
Omni-Realtime 模型响应延迟约1分钟，对于实时对话体验较差。

### 优化措施

#### 1. 速度模式切换
新增"快速模式"开关：
- **完整模式** (默认)：输出文本+语音，延迟约30-60秒
- **快速模式**：仅输出文本，延迟约3-5秒

#### 2. 音频分块优化
- 增大分块大小：从 16KB 提升到 48KB
- 减少分块间延迟：从 0ms 调整到 5ms（平衡效率和稳定性）

#### 3. 简化提示词
- 缩短系统提示词，减少模型处理时间

### 优化效果

| 模式 | 输出 | 预计延迟 |
|------|------|---------|
| 完整模式 | 文本+语音 | 30-60秒 |
| 快速模式 | 仅文本 | 3-5秒 |

### 使用方法
1. 在"小爱倾听"页面，勾选"快速模式（仅文本，无语音）"复选框
2. 点击"开始对话"
3. 录音并发送，观察响应速度
